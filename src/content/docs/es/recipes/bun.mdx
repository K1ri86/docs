[Bun](https://bun.sh/) es un conjunto de herramientas y tiempo de ejecución de JavaScript todo en uno. Consulte la [documentación de Bun](https://bun.sh/docs) para obtener más información.

:::caution
Bun recientemente realizó su primer lanzamiento estable. Sin embargo, usar Bun con Astro puede revelar asperezas. Es posible que algunas integraciones no funcionen como se esperaba. Consulte la [documentación oficial de Bun para trabajar con Astro](https://bun.sh/guides/ecosystem/astro) para obtener más detalles.

Si tiene algún problema al usar Bun, [abra un Issue en GitHub con Bun directamente] (https://github.com/oven-sh/bun/issues/new/choose).
:::

## Prerrequisitos

- Tener Bun instalado localmente en su máquina. Consulte las [instrucciones de instalación](https://bun.sh/docs/installation) en la documentación oficial de Bun.

## Crea un nuevo proyecto Astro con Bun

Cree un nuevo proyecto Astro con Bun usando el siguiente comando `create-astro`:

```bash
bunx create-astro@latest my-astro-project-using-bun
```

:::tip
También es posible [crear un nuevo proyecto de Astro a partir de cualquier repositorio de Astro en GitHub](/en/install/auto/#starter-templates) utilizando la opción `--template`:
```bash
bunx create-astro@latest my-astro-project-using-bun --template eliancodes/brutal
```
:::

## Instalar dependencias

Si está iniciando un nuevo proyecto usando `bunx create-astro`, la CLI usará automáticamente Bun para instalar dependencias tambien puede omitir este paso.

De lo contrario, necesitarás instalar tus dependencias con Bun:

```bash
bun install
```

## Usando integraciones de Astro

También puedes usar cualquiera de las integraciones oficiales de Astro con el comando `astro add`:

```bash
bunx astro add react
```

## Inicia Astro con Bun

:::note
Utiliza la opción de línea de comando [`--bun` ](https://bun.sh/docs/cli/bunx#shebangs) antes de cada comando `astro` para utilizar el tiempo de ejecución de Bun en lugar de Node, sin que afecte su funcionalidad al ser agregado al código.
:::

### Ejecute el servidor de desarrollo

Para ejecutar el servidor de desarrollo utilizando Bun como tiempo de ejecución, utilizar el siguiente comando:

```bash
bun --bun astro dev
```

### Construyendo su sitio con Bun

Para crear su sitio usando Bun como tiempo de ejecución, use el siguiente comando:

```bash
bun --bun astro build
```

Astro enviará su sitio al directorio `dist/`. Luego, puede usar su sitio usando el comando `preview`:

```bash
bun --bun astro preview
```

## Agregue SSR con Bun usando `@astrojs/node`

Bun ofrece [compatibilidad con la API de Node.js](https://bun.sh/docs/runtime/nodejs-apis). Al utilizar el adaptador [`@astrojs/node`](/en/guides/integrations-guide/node/), es posible añadir [renderizado del lado del servidor](/en/guides/server-side-rendering/) a su proyecto de Astro mediante el tiempo de ejecución de Bun, en sustitución de Node.

Ejecute el siguiente comando para agregar el adaptador Node.js a su proyecto Astro:

```bash
bunx astro add node
```

Construya su sitio nuevamente, usando el mismo comando de compilación anterior:

```bash
bun --bun astro build
```
Finalmente, use el siguiente comando para ejecutar su sitio creado.

```bash
bun ./dist/server/entry.mjs
```
